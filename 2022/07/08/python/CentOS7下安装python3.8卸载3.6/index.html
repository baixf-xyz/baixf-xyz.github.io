CentOS7下安装python3.8卸载3.6

在装好的kaldi的centos7上跑特征提取时发现一个问题，就是Librosa需要的python环境版本太低了，因为centos自带的是python36，所以索性卸掉python36,编译安装一下python38。

Librosa是一个 Python 模块，用于分析一般的音频信号，是一个非常强大的python语音信号处理的第三方库。

一、卸载

根据查到的资料，主要就是卸载，然后删除一些软连接删除干净，逻辑很简单，贴一些具体的操作代码，记录下来 。

卸载Python3的步骤

二、安装

1.安装依赖

如果有提示一路选择Y就可以。

安装libffi-devel依赖。

注意：如果不安装这个包，python3可以装成功，但是后面装flask、uwsgi等依赖python3中有个内置模块叫ctypes时会报错。报错ModuleNotFoundError: No module named ‘_ctypes‘。需要安装依赖包和重新编译安装python3。 参看：https://blog.csdn.net/qq_36416904/article/details/79316972

2.下载python源码包

下载稳定版本3.8版

3.解压python源码包

4.编译安装

进入解压后的目录进行编译和安装

执行完后显示安装成功

5.建立命令软链接

虽然python3.8.12安装成功了，输入python显示是2.7版本的，输入python3是找不到命令的，原因是因为没有把python3添加在环境变量中。为了实现两种python均保留在环境变量里，即要用python3.8.12需要输入python3即可，要用python2输入python。

可以通过修改软链接的方式将python3指向python3.8.12。
先看一下默认的python及新安装的python3都安装在哪里（那里应该就是环境变量位置）

可以看到默认的python路径为/usr/bin/python，python3的路径为/usr/local/bin/python3
在相同位置python3的软链接加到python上

通过python -V命令查看python版号，这时python的版本已经是3.8.12了。

6.添加pip3

pip命令也可以修改，python3.8.12默认的pip是pip3，CentOS7的python2.7默认没有安装pip。

输入pip命令的时候提示命令没有找到

这时也可以通过建立软链接的方式将pip命令链接到pip3上。首先看pip3命令在哪?

然后建立pip到pip3的软链接

7.配置yum*

如果你执意将python3的软连接设为python，那么恭喜你，安装python3改完软链接以后发现yum命令报错了，yum是依赖python2.7的，你把python改成了3.8了，所以报错了。

可以cd到/usr/bin目录下 通过ls -alh|grep python查看python命令的详细情况。

可以看到python软连接是执行的python2命令，python2是执行的python2.7的命令，上图为python3->python3.8，应该不会报错。

可以看到python软连接是执行的python3命令，python2是执行的python2.7的命令，这样你yum时会报错的。

以下针对错误的解决措施。

修改对python的依赖，修改成python2或python2.7都可以。

修改完这两个文件后，再敲yum命令就不会报错了（未试）。

至此CentOS7环境下python3.8.12已经成功安装！
